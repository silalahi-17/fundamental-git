Penjelasan Git

  • Git adalah salah satu version control system yang diciptakan oleh Linus Torvalds
  • Git mencatat setiap perubahan file di code atau source code kita

Manfaat

  • bisa menyimpan seluruh versi source code
  • Berkolaborasi membangun system menjadi mudah
  • bisa berkontribusi terhadap project open source
  • memudahkan dalam tracking perubahan
  • memahami cara deploy modern
  • Dsb

Kapan dibutuhkan

  • Saat ingin membantu system dalam Berkolaborasi
  • membuat versi dari sistem
  • Back Up source code
  • Berkolaborasi dengan system dunia

Deskripsi

  • Perintah dasar yang sering dilakukan untuk mengoperasikan git

Manfaat

  • Terbiasa mengerjakan proyek dengan terdokumentasi
  • Fleksibel dalam mengeksplorasi fitur
  • mempercepat dalam mengerjakan proyek terlebih jika proyek kolaborasi

Todo

  • Git init
    • digunakan untuk menginisialisasi lokasi folder sekarng dijadikan repository
  • Git config
    • untuk konfigurasi mendaftarkan username dan email (biasanya disamakan dengan akun github dan email yang didaftarkan)
  • Git status
    • digunakan untuk memeriksa status file ( terdapat 3 jenis status, yaitu : Untracked, Staging area, dan modified)
  • Git add
    • digunakan untuk menambahkan file ke staging area
  • Git commit
    • untuk mencatat perubahan yang dilakukan
  • Git clone
    • digunakan untuk mengklon repository
  • Git remote
    • digunakan untuk menghubungkan repository dengan remote repository
  • Git push
    • digunakan untuk mengirimkan perubahan ke remote repository
  • Git pull
    • digunakan untuk mengambil perubahan dari remote repository

Branch

merupakan cabang dari repository

Manfaat

  • Parallel development
  • Bisa digunakan untuk manajemen pengembangan. Misal test branch, unstable branch, dan stable branch

Perintah yang ada di branch

  • Git branch
    • melihat semua branch local
  • Git branch --all
    • melihat semua branch public
  • git branch name_branch
    • membuat branch baru
  • git checkout name_branch
    • berpindah ke branch yang sudah dibuat
  • git branch -d name_branch
    • menghapus branch yang sudah tidak digunakan

Git merge name_branch

  • untuk menggabungkan commit dari dua branch
  • Relatif terhadap branch saat ini
  • untuk melakukan merge dipastikan tidak ada file yang belum di commit

Conflict

  • kondisi yang terjadi ketika merging 2 branch yang memiliki perubahan yang sama
  • ketika Conflict, kita tidak bisa melakuan commit
  • untuk melakuan commit, conflict harus diselesaikan terlebih dahulu
